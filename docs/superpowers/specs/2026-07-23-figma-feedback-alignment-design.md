# KOOSIMY Figma 验收反馈对齐规格

## 目标与依据

在现有 React/Vite 静态网站中落实 Figma 文件 `Untitled` 的 `0722 修改反馈`，使首页、案例、关于、联系四个页面在 1920px 桌面视口下与设计稿保持一致，同时保证 1366px、1024px、768px 与 390px 宽度下不裁切 Banner、不遮挡图片、不出现横向溢出。

视觉优先级如下：

1. Figma 页面 `0722 修改反馈` 中的批注。
2. Figma 首页节点 `1:4`、案例节点 `1:253`、关于节点 `5:78`。
3. 本地 `首页.jpg`、`案例.jpg`、`关于.jpg`、`联系.jpg` 原型。
4. `0713切图` 中的原始位图与 SVG。

联系页设计上下文因 Figma Starter 套餐读取额度达到上限而无法再次导出，实施时使用本地 `联系.jpg` 与已读取的联系页批注作为精确基准，不使用临时 Figma 图片链接。

## 全站视觉系统

- 桌面设计基准宽度为 `1920px`。
- 桌面内容区最大宽度为 `1680px`，左右基准边距为 `120px`。
- 品牌红固定为 `#E50036`；不继续使用 `#E6002D`。
- 页头在桌面端高 `134px`，覆盖在 Banner 顶部，背景为约 `rgba(255, 255, 255, 0.8)` 的半透明白色。
- 桌面 Logo 按设计稿比例显示，最大宽度约 `582px`，左边距约 `58px`。
- 导航仅当前页面使用品牌红；鼠标悬停不得将普通导航文字变红。
- 中文字体优先级为 `MiSans`、`Source Han Sans SC`、`Noto Sans SC`、`Microsoft YaHei`、`PingFang SC`；代码中不再声明 Helvetica 或 Arial。
- 大标题、英文展示字优先使用 MiSans；正文优先使用思源黑体系列。
- 页脚桌面基准尺寸为 `1920 × 553px`，背景固定为 `#E50036`。
- 页脚补充图标授权标注：`Icons by Nobita (IcoFont), licensed under CC BY 4.0`，并链接到 `https://creativecommons.org/licenses/by/4.0/`。
- 所有图像使用 `0713切图` 的现有素材，不用 AI 生成图或 Figma 临时 URL。

## Banner 与响应式规则

- 首页、案例、关于、联系页 Banner 分别保持源图 `3840/2162`、`3840/1896`、`3840/2161`、`3840/2162` 的比例。
- Banner 图片必须完整可见，使用源比例容器与 `object-fit: contain`，浏览器变窄时不得使用固定高度或 `cover` 造成裁切。
- 页头绝对定位覆盖在 Banner 顶部，因此页头高度包含在设计稿首屏高度内，不额外推高 Banner。
- Banner 文案使用百分比位置与 `clamp()` 缩放；任意验收宽度下文字边界必须位于 Banner 内。
- 页面根节点在所有验收宽度满足 `scrollWidth === clientWidth`。
- `> 1100px` 保持设计稿桌面网格；`821–1100px` 采用收缩网格；`561–820px` 采用两列或单列；`<= 560px` 使用单列和移动导航。

## 首页

- Banner 背景不叠加灰色遮罩；左侧文案使用黑色，`IMAGINATION` 与“一站式产品设计研发服务”使用品牌红。
- Banner 文案位置、字距和行距按 Figma 1920 基准缩放，避免当前标题挤压。
- “我们能做什么”左侧对齐，右侧展示五项服务、竖向分隔线和原始 SVG 图标；服务卡不使用红色 hover 反转效果。
- 服务描述后补充“了解更多 →”入口。
- 工业设计项目区改为白底；前两张为两列大图，后三张为三列图。每项包含红色编号、中文标题、说明文案与“查看案例 →”。
- 品牌与数字项目区改为白底四列，每项包含序号、标题和“查看案例 →”。
- “荣誉与认可”标题左对齐；五个奖项 Logo 使用 K-Design、Red Dot、MUSE、IDA、iF 正确素材，不使用边框卡片。
- 缩短奖项区与“我们设计未来”横幅之间的空白。

## 案例页

- Banner 完整显示智能水壶双产品图。
- 项目介绍补齐设计稿中的全部段落，并包含结尾句：`它不仅是一台饮水设备，更像是一件安静陪伴生活的智能家居产品。在每一次接水、加热与等待之间，让日常变得更温柔、更有品质感。`
- 产品视觉区按设计稿呈现上方双图组合与下方大图，不增加设计稿之外的项目故事文案。
- 删除当前 `case-story` 标题与说明区。
- “查看更多案例”使用白色背景；两张图片完整显示，补充中文分类、中文标题与英文标题。
- 左右箭头使用设计稿中的细线箭头，不使用圆形按钮底框。

## 关于页

- Banner 不叠加灰色遮罩；文字使用黑色与品牌红，并补齐设计稿内容。
- 公司简介使用设计稿双栏比例与完整文案，右侧工作室图片为 `789 × 549` 的桌面基准比例。
- 数据区保留 `8+ / 100+ / 300+ / 1`，缩短上下巨大空白。
- 五项服务能力按细分隔线列表布局；桌面单行高度约 `188px`，补齐设计稿说明。
- 两位创始人使用约 `1532 × 462px` 的横向紧凑卡片，不使用当前超高 `610px` 卡片与巨大间距。
- 团队区和合作伙伴区均为白色背景；团队桌面五列两行，合作伙伴 Logo 不使用边框卡片。
- 合作伙伴说明使用设计稿完整文案：`经过多年的发展，酷施美设计已成功携手上海电气、西屋、大宇等超200家国内外知名企业和新兴互联网品牌，打造出1000余个优秀案例。我们擅长为企业量身定制人性化的产品规划与设计服务，用严谨创新、精益求精的设计助力成就产品和品牌价值`。

## 联系页

- 按 `联系.jpg` 保持 Banner、联系方法、项目咨询、到访工作室、多平台触达、未来横幅和页脚的顺序。
- 修正当前 AI 风格的大留白和错位版式。
- 联系方法区保留五项原始 SVG 图标，并补齐设计稿大标题。
- “有想法？我们很乐意倾听”使用左侧文字、右侧咨询图片的桌面构图。
- 地图与地址信息按原型比例显示，地图不得被异常拉高。
- 多平台触达区为白底，四个品牌素材保持原图比例。

## 可访问性与交互

- 导航、电话、邮箱、项目咨询与案例入口使用语义化链接。
- 移动导航按钮与案例轮播按钮保留至少 `44 × 44px` 触控区域和可读的 `aria-label`。
- 所有信息性图片提供中文替代文本，纯装饰 SVG 使用空替代文本。
- 动画继续尊重 `prefers-reduced-motion`。

## 自动化与视觉验收

- 使用 Vitest/Testing Library 验证关键文案、设计稿要求的 DOM 结构、删除的额外案例故事、相关推荐英文标题和页脚授权。
- 使用 CSS 契约测试验证品牌红、1680 内容宽、134 页头、透明背景、导航 hover、字体栈、Banner 不裁切和 553 页脚比例。
- 完成后运行 `npm run check`。
- 在 `/`、`/cases`、`/about`、`/contact` 的 1366、1024、768、390px 宽度逐页检查 Banner 比例、文字边界、图片完整性和横向溢出。
- 生成桌面运行截图与本地原型的对比图，作为最终人工复核依据。
