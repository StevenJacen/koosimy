# KOOSIMY 响应式 Banner 修正设计

## 背景

当前四个页面使用了三套 Banner 高度规则：首页依赖视口高度，案例页依赖 `74vw`，关于页与联系页依赖 `62vw`，移动端又统一改为固定高度。所有图片同时使用 `object-fit: cover`。容器比例和素材比例不一致时，浏览器只能通过裁切图片填满容器，因此窗口变窄后会遮住图片主体。

## 目标

- 每个 Banner 按对应原型素材的原始比例缩放。
- 浏览器宽度变化时完整展示图片，不横向或纵向裁切。
- 四个页面使用同一套结构与样式契约，避免再次出现不同高度公式。
- 小屏上的覆盖文案同步缩放，保持在 Banner 内且不遮挡主要图片内容。

## 方案

新增共享的 `ResponsiveHero` 组件。组件负责输出统一的 `responsive-hero` 容器、媒体图片及可选覆盖内容，并通过 CSS 自定义属性接收素材宽高比。

各页面使用自己的素材比例：

- 首页：`3840 / 2162`
- 案例页：`3840 / 1896`
- 关于页：`3840 / 2161`
- 联系页：`3840 / 2162`

容器使用 `aspect-ratio`，图片使用 `object-fit: contain`。这样案例页可以保留原型中更宽、更短的 Banner，而其他页面保持接近 16:9 的比例；不同页面不再共享错误的固定像素高度。

## 响应式行为

- 桌面端：按页面宽度和素材比例自动计算高度。
- 平板端：继续按比例缩放，不设置固定高度。
- 手机端：保持完整图片，缩小覆盖标题、说明、内边距和底部服务文案，使内容在较矮的等比 Banner 内可读。
- Banner 图片背景色使用各页面现有底色，防止因子像素取整出现可见缝隙。

## 测试与验证

- 组件测试验证四页都使用共享的响应式 Hero 契约和正确宽高比。
- 样式契约测试验证 Banner 使用 `aspect-ratio` 与 `object-fit: contain`，且移动端不再设置固定高度。
- 浏览器在 1366、1024、768、390 四种宽度下验证四个页面：Banner 比例正确、裁切比例为 0、无横向溢出、覆盖文案不超出容器。
