# KOOSIMY 静态网站设计规格

## 目标

在现有 React/Vite 项目内重建 KOOSIMY 企业官网，使首页、案例、关于、联系四个页面在 1920px 桌面视口下严格遵循 `首页.jpg`、`案例.jpg`、`关于.jpg`、`联系.jpg` 的视觉结构，并使用 `0713切图` 中的原始素材。网站同时提供可用的平板和手机布局。

## 验收基准

- 四张原型图是唯一视觉基准，现有页面实现不构成约束。
- 桌面端按原型的栏目顺序、留白、网格、颜色、字体层级和图片裁切方式还原。
- 案例页是“玉环职智能水壶”单一案例详情页，不扩展为案例列表页。
- 中文内容必须正常显示，不保留乱码、占位图或无效图片引用。
- 导航、轮播按钮、电话、邮箱和返回顶部等可见交互应可使用。
- 生产构建可在静态托管环境运行，刷新任意路由后仍能正常显示。

## 信息架构

网站包含四条路由：

- `/`：首页
- `/cases`：案例详情
- `/about`：关于我们
- `/contact`：联系我们

所有路由共用顶部导航、页尾行动横幅和红色页脚。路由变化时页面滚动位置重置到顶部。

## 视觉系统

- 主色：高饱和 KOOSIMY 红，接近 `#E6002D`。
- 中性色：纯白背景、接近 `#151515` 的正文黑、浅灰分隔线。
- 中文字体优先使用 `Noto Sans SC`、`Microsoft YaHei`；英文标题使用 `Arial`/`Helvetica` 系无衬线粗体。
- 内容最大宽度约 1440px；大图区域可全宽铺陈。
- 页面以大留白、细分隔线、红色英文标签、黑色中文标题组成统一节奏。
- 动效仅用于首屏淡入、滚动显现和案例切换，不改变原型的静态构图。

5. 品牌与数字项目四列网格。
6. 荣誉与认可 Logo 横排。
7. “WE DESIGN THE FUTURE” 共同行动横幅。
8. 红色页脚。

7. 合作伙伴 Logo 墙。
8. 公共行动横幅与页脚。

### 联系

## 组件边界

- `SiteHeader`：品牌标志、桌面导航、移动菜单。
- `SiteFooter`：地址、电话、邮箱、二维码。
- `FutureBanner`：四页复用的底部品牌横幅。
- `SectionHeading`：英文标签、中文标题和红色短线。
- `Reveal`：尊重 `prefers-reduced-motion` 的进入动画容器。
- 页面文件只负责各自的数据和布局，公共内容不重复实现。

## 素材策略

- 从 `0713切图/0713切图` 选取原型对应素材并复制到 `public/assets`。
- 文件统一改为稳定的英文 kebab-case 名称，代码中不引用包含空格或中文的源文件名。
- 对大幅 JPG 使用源文件并通过 CSS 控制裁切；首屏图片优先加载，其余图片懒加载。
- 使用素材中的官方 SVG Logo 和图标；不使用现有错误占位图。
- SVG 映射：
  - `酷施美logo .svg` → 全站导航 Logo。
  - `wodeyanjiu 1.svg`、`jiaohusheji 1.svg`、`weibiaoti-4 1.svg` → 首页策略研究、工业设计、供应链支持图标。
  - `huizhi-01 1.svg`、`Group 18.svg` → 首页 K-DESIGN 与 RED DOT 奖项标志。
  - 联系页 `dianhuazixun 1.svg`、`Group 42.svg`、`Group 40.svg`、`Group 46.svg` → 电话、邮箱、地址、商务合作图标。

## 响应式行为

- `>= 1280px`：按原型的桌面网格和大留白布局。
- `768px–1279px`：两列为主，保持栏目顺序和图片比例。
- `< 768px`：单列布局、抽屉式导航、横向可滚动的合作伙伴和案例区域。
- 标题使用 `clamp()` 缩放，避免截断；触控目标不小于 44px。

## 测试与验证

- 使用 Vitest 与 Testing Library 验证四条路由、导航活动状态、关键文案和案例切换。
- 运行 ESLint、TypeScript 和 Vite 生产构建。
- 在桌面与移动视口截图，逐页对照原型检查信息层级、图片、间距、溢出和乱码。
- 检查所有本地图片请求、电话和邮箱链接。

## 发布

- 在 `app` 目录初始化 Git 仓库，默认分支为 `main`。
- 绑定 `https://github.com/StevenJacen/koosimy.git`。
- 完成验证后提交全部网站文件并推送到远程 `main`。
